## Quick Setup (2 minutes)

### Step 1: Build Delegate
```bash
cd /workspace/delegate
npm run build
```

### Step 2: Configure MCP

**Option A: Automatic Setup**
```bash
./setup-mcp.sh
```

**Option B: Manual Setup**

1. Check if config exists:
```bash
cat ~/.config/claude/mcp_servers.json
```

2. If it exists, ADD this to it:
```json
{
  "mcpServers": {
    "delegate": {
      "command": "node",
      "args": ["/workspace/delegate/dist/index.js"],
      "env": {
        "MOCK_MODE": "true"
      }
    }
  }
}
```

3. If it doesn't exist, create it:
```bash
mkdir -p ~/.config/claude
cp /workspace/delegate/config/mcp-config-ready.json ~/.config/claude/mcp_servers.json
```

### Step 3: Start NEW Claude Code Session

**Important**: Must be a NEW session, not `--continue`
```bash
# Exit current session
exit

# Start fresh session
claude "Let's test Delegate MCP tools"
```

### Step 4: Test Integration

In the new Claude Code session, try:

1. **Check available tools**:
   > "What MCP tools are available?"

2. **Test delegation**:
   > "Use DelegateTask to run: echo 'Delegate is working!'"

3. **Check status**:
   > "Use TaskStatus to check the current task"

## Expected Behavior

When working correctly, you should see:
- 4 tools available: DelegateTask, TaskStatus, TaskLogs, CancelTask
- Task IDs in UUID format
- Mock output (since MOCK_MODE=true)

## Troubleshooting

### "Tools not found"
- Ensure NEW session (not --continue)
- Check config exists: `ls ~/.config/claude/mcp_servers.json`
- Verify path is correct: `/workspace/delegate/dist/index.js`

### "Server won't start"
```bash
# Test directly
MOCK_MODE=true node /workspace/delegate/dist/index.js

# Should see: "Delegate MCP Server running"
```

### "Build errors"
```bash
npm run clean
npm install
npm run build
```

## Using Real Claude CLI

To use with actual Claude CLI (not mock):

1. Remove `"MOCK_MODE": "true"` from config
2. Ensure `claude` CLI is installed
3. Test with real tasks

## What You Can Do Now

With Delegate running, you can:
